# King of the Hill

Compete in a two player game of chess that uses a special variant of chess.

What happens when the game can be won by being on the proverbial "hill"? 
As long as a King piece is moved to be in the legal four squares at the center of the board, the game can be won.

The lessons I learned from developing this project include but are not limited to:
* the memory required to remember and follow logic that requires many levels of nested booleans and for loops
* to utilize and plan detailed pseudocode before the real coding begins
* the importance of properly documenting function intent with docstrings
* to consider edge cases
* to test often

Possible future considerations:

As someone who now reviews this code with new eyes, I see the potential for Test Driven Development for this sort of project. I also think possibly offering a "computer" opponent would make this an even more robust project.
Creating a UI with images would enrich the user's experience.
